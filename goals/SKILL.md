# Mochary Method: Goals

Create, review, and coach goals using the Mochary Method. For full detail including the coaching flow and anti-patterns table, read `references/goals-full.md`.

## Creating Goals

1. **Outcome-first:** "What result would make me say 'I crushed it'?"
2. **Time-bound:** Shorter is better. Ideal: 30-day goals.
3. **Prioritized:** If you could only do ONE, which? That's #1.
4. **Limited:** Max 3 work goals + 1 personal. Ideally just 1.

## Creating Next Actions

Every action must pass the **Input Test:** "Can I complete this 100% on my own?"

| Output (bad) | Input (good) |
|---|---|
| "Close customer X" | "Send proposal to customer X" |
| "Get approval from boss" | "Send brief with recommendation by Tuesday" |
| "Wait for vendor" | "Send follow-up with Friday deadline" |

Every action needs: input framing + due date + specificity.

### The 2-Minute Rule

- Action < 2 min → do it NOW
- Action > 2 min → "Schedule the time to do it" (that scheduling takes < 2 min, so do THAT now)

## Weekly Review Flow

For each goal:

1. **Review prior actions:** Done → learnings? Not done → what blocked you? Still want to? By when? What changes to avoid same block?
2. **Status:** On Track / At Risk / Off Track / Complete
3. **Good:** What's going well
4. **Not Good:** What's not going well (where coaching value lives)
5. **New actions:** Input-framed, time-bound, specific. Apply 2-minute rule.
6. **Confirm:** Read back full update before recording

## Coaching Principles

- **One question per message.** Don't overwhelm.
- **Actions, not intentions.** "I'll try to improve X" → "What specific step will you take, and by when?"
- **The intention is not punishment.** Missed actions reveal system weaknesses to fix, not personal failures to judge.
- **Probe "On Track."** If status is always green, ask: "What would make this Off Track? How close are you?"
- **Fewer goals, more focus.** Push toward 1 goal when possible. 3 is the max.

## Gotchas

- **"Continue doing X" is not an action.** It has no endpoint, no due date, and no way to mark it done. Reframe to a specific next step.
- **Outputs masquerade as inputs.** "Improve retention" sounds controllable but isn't. The input version: "Analyze churn data and draft 3 recommendations by Friday."
- **Due dates must be calendar dates.** Not "within 48 hours" or "soon" or "next week." "By Thursday, March 20" is the format.
- **Too many actions = none get done.** If someone proposes 7 actions, ask: "Which ONE matters most this week?" Start there.
- **Past-due goals are Off Track by definition.** Don't let someone call a missed-deadline goal "At Risk." If it's past due and not done, it's Off Track.
- **"I didn't have time" is always a prioritization problem, not a time problem.** Coach the prioritization, don't accept the excuse.
