# Goals: Full Reference

Source: Mochary Method (Matt Mochary)
Docs: Goals (how to create them), Goals (how to review them), Actions (how to review), First 2 Minutes of Actions

## 1. Creating Goals

### Outcome-First

Ask: "What outcome, if it occurred, would make me say 'I crushed it in my role'?"

### Rules

- **Time-bound:** Week, month, quarter, or year. Shorter = more action-oriented = better. Ideal: 30-day goals.
- **Prioritized:** If you could only get ONE done, which is it? That's #1.
- **Limited:** Never more than 3 work goals + 1 personal. Best: track only 1 goal for true focus.

### Good vs Bad Goals

| Good | Bad |
|---|---|
| "Achieve $150 CAC on a $79 product by end of Q1" | "Improve marketing" (vague, no outcome) |
| "Launch presell page by Feb 28" | "Work on optimization" (activity, not outcome) |
| "Hire VP Engineering by March 15" | "Find good candidates" (no measurable endpoint) |

## 2. Creating Next Actions

### The Input Rule

Actions must be **inputs** (things you control entirely), not **outputs** (things that depend on others).

Ask: "Can I complete this 100% on my own, without anyone else deciding or acting?"

| Output (bad) | Input (good) |
|---|---|
| "Close customer X" | "Send proposal to customer X" |
| "Get approval from boss" | "Send boss the brief with my recommendation by Tuesday" |
| "Improve retention" | "Analyze M1 churn cohort data and draft 3 recommendations" |
| "Wait for vendor response" | "Send follow-up email to vendor with deadline of Friday" |

### The 2-Minute Rule

Make the action's timeframe as short as possible. Ask: "What can I do in the next 2 minutes?"

- Action takes >2 min → the 2-minute version is "schedule the time to do it"
- Action takes <2 min → do it NOW

**Example:**
- "Send a proposal to X customer" → requires 2 hours
- 2-minute version: "Schedule 2 hours on my calendar to write and send the proposal"
- This takes <2 min. Do it immediately. The flywheel starts.

### Every Action Must Have

1. **A due date** — no exceptions
2. **Input framing** — something the person fully controls
3. **Specificity** — what exactly will be done, not vague intent

## 3. Bias to Action: First 2 Minutes

From Frank Slootman's *Amp It Up*: When someone says "I'll do this by next week," challenge: "Why not tomorrow?" or "Why not right now?"

Reserve the last 10 minutes of every meeting to begin each action:

**Actions you do yourself:**
- < 2 min → do it immediately
- \> 2 min → estimate time, create a calendar block with the action as title

**Actions involving someone else:**
- Already have a meeting scheduled → add to that meeting's agenda
- No meeting: message takes < 2 min → send it now
- No meeting: message takes > 2 min → schedule calendar time to compose and send

Starting eliminates the hardest part (beginning). Small beginnings create self-sustaining momentum.

## 4. Reviewing Goals (Weekly)

For each goal, the weekly update includes:

### Status
- **On Track** — progress as expected
- **At Risk** — something is threatening progress
- **Off Track** — behind, needs intervention
- **Complete** — done

Past-due goals that aren't completed = Off Track by definition.

### Good
What's going well. Wins, progress, momentum.

### Not Good
What's not going well. Honesty here is where the coaching value lives.

### Prior Actions Review

For each action declared last week:

**If DONE:**
- Any key learnings?
- Any follow-up actions to spawn?

**If NOT DONE:**
- What blocked you?
- Do you still want to do this? If yes, by when?
- What will you do to prevent the same block? (This becomes a new action with a due date)

**The intention is NOT punishment.** It's building a system that lets people consistently complete priority actions. Blockages are opportunities to strengthen the system.

### New Actions

Actions for the coming week. Each must be:
- An **input** (fully in their control)
- **Time-bound** (has a due date)
- **Specific** (not vague intent)
- Ideally scoped to 2 minutes, or the 2-minute version is "schedule time to do it"

## 5. Coaching Flow

When coaching someone through a goal review session:

### Step 1: Review Prior Actions
- For each: "Did you complete [action]? Done or not done?"
- If done: "Any learnings or follow-up actions?"
- If not done: "What blocked you? Do you still want to do it? By when? What will you change to avoid the same block?"

### Step 2: Status + Good + Not Good
- "What's the status? On Track, At Risk, or Off Track?"
- "What's going well?"
- "What's not going well?"

### Step 3: New Actions
- For each proposed action, apply the input test
- Ensure each has a due date
- If < 2 min: "Can you do that right now?"
- If > 2 min: "When will you schedule time for this?"

### Step 4: Confirm
Read back the full update for confirmation before recording/posting.

## 6. Common Anti-Patterns

| Anti-Pattern | How to Coach |
|---|---|
| "Continue doing X" | Not an action. What specific next step will you take? |
| "Wait for X to happen" | That's an output. What can YOU do to move this forward? |
| "Improve X" | Too vague. What's the first concrete step? |
| No due date | "By when?" Every action needs a date. |
| Action depends on someone else deciding | Reframe: "Send [person] [specific thing] by [date]" |
| Too many actions (>5) | Prioritize. What's the ONE thing that matters most? |
| Status always "On Track" | Probe: "What would make this Off Track? How close are you to that?" |
| Goal is an activity, not an outcome | "What result would this activity produce? That's your real goal." |
