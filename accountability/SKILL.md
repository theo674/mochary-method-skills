# Mochary Method: Accountability

Eight interconnected frameworks for execution discipline. For full detail, read `references/accountability-full.md`.

## Framework Selection Guide

| Situation | Framework |
|---|---|
| Actions agreed upon but never completed | **Agreement Tracker** |
| Nobody owns a metric, project, or goal | **DRIs** |
| Cross-team conflict, one-sided complaints to CEO | **Clean Escalation** |
| Tasks keep getting deferred, no momentum | **First 2 Minutes of Actions** |
| Functions falling through cracks, no backup coverage | **AORs** |
| Need quarterly goals with buy-in | **OKR Creation** |
| Procrastinating on admin/unfun tasks | **Accountability Partner** |
| Creating goals that actually drive action | **Goals: Create and Review** |

## Quick Reference

### Agreement Tracker
One place per person to see all agreed-upon actions. Use Asana/Monday/ClickUp/Notion, not scattered Google Docs. Only the doer writes the task (never blindly assign). Roll out: yourself → 1-1s → team meetings → cascade company-wide.

### DRIs (Directly-Responsible Individuals)
Every action, metric, goal, project, meeting, and decision needs exactly one DRI. The DRI reports status AND pushes toward completion. Someone as DRI is always better than no one. Junior DRI can't compel action? They Clean Escalate to the Apex Manager.

### Clean Escalation
When two parties disagree: both prepare written positions → jointly request meeting with Apex Manager → Apex reads both, asks questions, decides. One mulligan allowed. **Say "no" to all dirty escalations** (one party complaining alone).

### First 2 Minutes of Actions
The hardest part is starting. Reserve last 10 min of every meeting to begin each action:
- < 2 min → do it now
- \> 2 min → schedule calendar time
- Involves someone else → add to next meeting agenda or message now

### AORs (Areas of Responsibility)
List every function, assign one owner each. No shared ownership (tragedy of the commons). Cross-train a backup for each role. Write down processes the second time you do anything.

### OKR Creation
3 Objectives, 3 Key Results each. Company → department → team → individual. CEO decides (not consensus) after structured brainstorming. Keep creation to one Admin Week. Track weekly with Green/Yellow/Red. All Red OKRs get a written Issue/Solution.

### Accountability Partner
For tasks you hate (usually admin): have another human simply be present while you do it. EA, coworker, or Focusmate. Especially effective for extroverts.

### Goals: Create and Review
1. Outcome-first: "What would make me say I crushed it?"
2. Max 3 work goals + 1 personal (ideally just 1)
3. Next actions must be **inputs** (things you control), not outputs
4. Break to < 2 min → do immediately → momentum flywheel begins

## Coaching Application

When helping someone with execution/accountability:
1. Are actions being tracked? If not → introduce Agreement Tracker
2. Does every metric/project/goal have a named owner? If not → assign DRIs
3. Are cross-team conflicts festering? → teach Clean Escalation
4. Are tasks perpetually deferred? → apply First 2 Minutes
5. Are functions falling through cracks? → build AOR list
6. Do they need quarterly goals? → run OKR creation process
7. Are they stuck on admin? → suggest Accountability Partner
8. Are goals fuzzy or output-focused? → rewrite as input-driven actions

## Gotchas

- **Don't blindly assign tasks in the tracker.** Only the doer writes the task. If you assign it, they never agreed to it and won't own it.
- **Agreement Tracker ≠ personal to-do list.** Only track actions agreed upon with others. Personal tasks go in a separate tool or the tracker becomes overwhelming.
- **DRI doesn't mean decision-maker.** The DRI reports status and pushes progress. Their manager is the decision-maker on direction. Confusing the two causes either paralysis or rogue decisions.
- **Dirty escalations are addictive.** Once you say "no" to one-sided complaints, people will test you. Hold the line. Clean Escalation only works if dirty ones are consistently rejected.
- **OKRs without company-level OKRs first creates silos.** Department OKRs that don't cascade from company OKRs lead to fiefdoms that don't trust each other.
- **"First 2 minutes" doesn't mean do the whole task.** It means start it (open the doc, send the scheduling email, write the first line). Starting is the hard part; momentum handles the rest.
- **Goals must be inputs, not outputs.** "Close customer X" is an output you don't control. "Send proposal to customer X" is an input you do.
